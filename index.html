HTML

  • Enter your HTML notes here
  • The head element contains information about the webpage
  • The body element contains visible content shown to the user
  • Within the body element, several tags are used to order information in a way that reads well to the user
  • HTML itself is largely used to structure webpages in ways that make them coherent to both browser and user, but does not add any style or logic to the page.
  • The ways it does this is by employing headers, sections, images, and other ordinal devices to give the page a skeleton off which to build a fuller "body" (hehehe).

CSS

  • Enter your CSS notes here
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS is used to ass style to HTML sites either by inline CSS (writing in style to an existing piece of HTML code), internal style sheets (in which rules for style are written into the "head" element of a page), or most commonly an external style sheet that holds the style rules and is linked to in the "head" element.

Git

  • Enter your Git notes here
  • To check what branch you're working on, type "git status" into the CLI
  • To start a new branch, type "git checkout -b [your branch name here]" into the CLI
  • The "git checkout" command is used to move between branches, and "-b" is used to create a new branch
  • To add modified files to the branch, you'll use the "git add" command, followed by the file name, or "-A" if you're looking to add all files from a directory
  • Using the "-m" command assogit ciates a message with any commit you make

JavaScript

  • Enter your JavaScript notes here
  • JavaScript is a programming language that uses logical statements to create functions and interactivity on a webpage.
  • Unlike HTML and CSS, which use drab but otherwise simple syntax to order the appearance of information ON a webpage, JavaScript creates processes and events in order to allow users to affect the webpage themselves (within defined parameters, of course)
  • As this is the most complex language we've looked at thus far in the pre-work, we'll be spending a lot of time on it in the BootCamp proper, so don't let it overwhelm you. You'll have time to learn what you need as you go.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Functions are processes that are "called" by developers and user input to make sure that they're infinitely usable and controlled.
